## Demo Flow

1. Log in as Admin.
2. Open Tournament.
3. Expand the Race and RaceEntry section.
4. Use a Race with 2 to 6 assigned RaceEntry rows.
5. Click `Khởi chạy Race`.
6. Backend generates a per-launch token and launches Unity if `UNITY_EXECUTABLE_PATH` is configured.
7. Unity fetches lineup from:

```text
GET /api/race-engine/{raceId}/lineup
```

8. Unity streams live ticks to:

```text
POST /api/race-engine/{raceId}/tick
```

9. Unity posts final result to:

```text
POST /api/race-engine/{raceId}/result
```

10. Backend saves `RaceResult` rows, marks Race as `COMPLETED`, clears the launch token, and the Admin UI updates.

## Manual Unity Mode

If Unity is opened manually from the Editor:

1. Click `Khởi chạy Race` in the Admin UI.
2. Copy the token from the launch popup.
3. Paste it into the Unity Token field.
4. Enter or confirm the Race ID.
5. Start the race in Unity.

## Common Issues

`UNITY_EXECUTABLE_PATH` is wrong:

- Backend marks the Race live but cannot start Unity automatically.
- Fix the path and launch again with another eligible Race.

Race has more than 6 entries:

- Unity demo scene supports up to 6 horses.
- Use a Race with 2 to 6 assigned entries.

Race is stuck `IN_PROGRESS`:

- Use the Admin recovery action to mark the launched Race as failed/cancelled with a reason.

Token error from Unity:

- Use the fresh token returned by the latest `Khởi chạy Race` action.
- Old tokens are cleared after completion/cancellation.

Database missing Race Engine columns:

- Recreate DB from `backend/src/main/resources/db/team_schema.sql`, or run `backend/src/main/resources/db/race_engine_migration.sql` on an existing DB.
